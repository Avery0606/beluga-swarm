# 错误示例：常见 AGENTS.md 问题

## ❌ 问题一：缺少必要章节

```markdown
# 我的项目

## 开发约束
- 必须做类型检查

## 项目开发指南
- 使用 TypeScript
```

**问题**：缺少"项目简介"、"关键架构列表"等必要章节，AI 无法理解项目背景和结构。

---

## ❌ 问题二：MUST/MUST NOT 标签缺失或位置错误

```markdown
# 我的项目

## 项目简介
这是一个代码开发插件

## 开发约束
- 必须做类型检查（错误：缺少 [MUST] 标签）
- 禁止修改 node_modules（错误：缺少 [MUST NOT] 标签）
- 建议使用 TypeScript（错误：不应出现在开发约束中）
```

**问题**：开发约束章节每条必须使用 RFC 2119 关键词标签（`[MUST]` / `[MUST NOT]` / `[SHOULD]` 等），不得出现无标签的约束；`[SHOULD]` 级别的建议内容如需体现，也应使用对应标签。

---

## ❌ 问题三：关键架构描述过于模糊

```markdown
## 关键架构列表
src/
├── index.ts    # 源码
├── tests/      # 测试
└── docs/       # 文档
```

**问题**：只描述了目录名称，没有说明业务背景和功能。AI 无法理解各目录的职责和关系。

---

## ❌ 问题四：格式不规范

```markdown
# 项目简介
这是一个项目

# 关键架构
src/：源码

## 开发约束
- MUST: 必须做类型检查

## 注意事项
一些注意事项
```

**问题**：标题层级混乱（有的用一级标题，有的用二级标题），应严格遵循模板的章节顺序和层级。

---

## ❌ 问题五：根目录 AGENTS.md 缺少项目开发指南

```markdown
# 我的项目

## 项目简介
这是一个代码开发插件

## 关键架构列表
src/    # 源码目录

## 开发约束
- [MUST] 必须做类型检查
```

**问题**：根目录 AGENTS.md 必须包含"项目开发指南"章节，说明项目的开发手段和注意事项。子目录 AGENTS.md 则不需要此章节。
